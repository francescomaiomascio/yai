# C17.2 Docs Naming Audit

## Scope

Audit of naming quality across live documentation (`docs/**` excluding `docs/archive/**`) after C17.1 topology refoundation.

## Key Findings

- Live docs were still using phase-oriented and migration-era suffixes.
- High-noise suffix families before cutover included: `model`, `baseline`, `closeout`, `refoundation`, `legacy-notes`, `residue`.
- Architecture used inconsistent terminal classes (`*-model`, `*-flow`, `*-topology`) without explicit grammar.
- Program IDs used mixed uppercase code forms (`ADR-*`, `RFC-*`, `MP-*`) with uneven lexical rules.
- Compound architecture folder names were still transitional.

## Live Suffix Inventory (pre-cutover)

- `model`: dominant in architecture and governance narrative docs.
- `baseline`: dominant in runbook/program operational material.
- `closeout`: present in milestone/report-like material.
- `refoundation`: transitional, phase-specific suffix.
- `legacy-notes`: appeared in live guides.
- `residue`: transitional naming residue in workspace docs.

## Live Suffix Inventory (post-cutover)

Disallowed suffixes removed from live docs:
- `model`, `baseline`, `closeout`, `refoundation`, `legacy-notes`, `residue`

Canonical suffixes now used:
- architecture family: `architecture`, `topology`, `flow`, `state`, `boundary`, `integration`
- guides family: `guide`, `workflow`, `walkthrough`, `checklist`, `debugging`
- runbooks family: `runbook`, `procedure`, `qualification`, `expected-outputs`
- reference family: `reference`, `contracts`, `schemas`, `registries`, `commands`, `sdk`
- program family: `rfc`, `adr`, `milestone-pack`, `policy`, `report`, `matrix`

## Archive Scope

Archive renames were minimized. Historical naming remains where useful for traceability, except paths referenced directly by live docs.
