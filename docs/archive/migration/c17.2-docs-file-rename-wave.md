# C17.2 Docs File Rename Wave

## Wave Summary

Policy-driven rename cutover across live docs completed with file-by-file mapping.

## Scope

- Architecture naming standardization
- Guides/runbooks/reference lexical normalization
- Program ID migration (`RFC`/`ADR`/`MP` to semantic lowercase grammar)
- Link and reference rewrites

## Volume

- Renamed files: 200+ (live docs + program IDs)
- Target threshold exceeded (minimum 80)

## Mapping Rules Applied

1. `*-model.md` -> class-specific terminal (`architecture/topology/flow/state/boundary/integration`)
2. `*-baseline.md` -> `*-runbook.md` or `*-qualification.md` by family
3. `*-closeout.md` -> `*-report.md`
4. `*-refoundation.md` -> `*-architecture.md`
5. `*-legacy-notes.md` -> moved to archive or renamed as archive note
6. RFC/ADR/MP legacy coded names -> semantic-ID grammar

## Link Rewrite Scope

Updated references in:
- `docs/**` markdown links and cross-links
- docs README spines and section contracts
- docs-related validators/tool scripts where path-sensitive
